Integración de Microservicios (Compras, Inventario, Facturación)

Este Módulo de Seguridad es el **núcleo de autenticación centralizado** de la universidad. Las aplicaciones independientes (desarrolladas por tus compañeros en React y NestJS) deben utilizar la siguiente lógica para asegurar sus rutas y validar a sus usuarios.

1. Inicio de Sesión Centralizado (GraphQL Mutation)

Tus compañeros deben enviar una consulta POST HTTP al endpoint /graphql/ con la siguiente estructura de mutación. Si las credenciales y el estado del usuario/módulo son válidos, la respuesta retornará el token JWT.

mutation { login(username: "nombre_usuario", password: "password_usuario", moduloId: 1) { success token message } }
2. Verificación de Permisos localmente (JWT Payload)

El JWT devuelto contiene la información del usuario, sus roles y sus funciones asignadas. Los microservicios externos pueden decodificar el token localmente (usando la misma llave secreta) y validar los permisos en su backend (por ejemplo en NestJS Guards):

{ "user_id": 4, "user_name": "david", "email": "david@example.com", "roles": ["Administrador", "Compras"], "permissions": ["CREAR_ORDEN", "APROBAR_ORDEN", "LEER_USUARIOS"], "exp": 1781329333 }
3. Consulta del Usuario en Sesión (GraphQL Query)

Si un microservicio necesita consultar los datos del usuario en tiempo real, puede enviar el token en la cabecera HTTP Authorization: Bearer <TOKEN> y ejecutar el query me:

query { me { id userName email estado roles { nombreRol funciones { nombreFuncion } } } }
